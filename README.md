# Scaler Clone Agent — Conversational AI CLI

An interactive command-line AI agent (similar to Cursor or Windsurf) that takes natural-language instructions, reasons through them in a `START → THINK → TOOL → OBSERVE → OUTPUT` loop, and writes real files to disk.

The flagship use case: ask it to "clone the Scaler Academy website" and it will create a working `scaler-clone/` folder with `index.html`, `style.css`, and `script.js` that visually resemble [scaler.com](https://www.scaler.com) — complete with header, hero, and footer.

---

## Features

- **Conversational CLI** — chat with the agent in your terminal; keep iterating ("make hero darker", "add a testimonials section").
- **Reasoning loop** — the model emits one JSON step at a time, the host executes the tool, feeds the observation back, and the loop continues until an `OUTPUT` step is reached.
- **Seven tools the agent can call:**
  - `writeFile({ path, content })` — write text files (creates parent dirs)
  - `readFile(path)` — read a file (truncated at 4 KB)
  - `createDirectory(path)`
  - `listDirectory(path)`
  - `executeCommand(cmd)` — run a shell command and capture stdout
  - `getTheWeatherOfCity(city)`
  - `getGithubDetailsAboutUser(username)`
- **Free LLM backend** — talks to Google Gemini through its OpenAI-compatible endpoint (free tier from [aistudio.google.com](https://aistudio.google.com)).
- **Rate-limit safe** — built-in throttle (`MIN_DELAY_MS`) plus exponential-backoff retry on 429s, so free-tier limits don't break the run.
- **Strict JSON output** — uses `response_format: { type: "json_object" }` and recovers gracefully if the model ever drifts.

## Setup

```bash
git clone <your-repo-url>
cd scaler-clone-agent
npm install
cp .env.example .env
# open .env and paste your Gemini API key
npm start
```

You only need a free key from [aistudio.google.com](https://aistudio.google.com) (no card required).

### Switching models

Default model is `gemini-2.5-flash`. To swap, add a line to `.env`:

```
MODEL=gemini-2.0-flash
MIN_DELAY_MS=4000
```

`MIN_DELAY_MS` is the minimum gap between API calls (default `6500` ms ≈ 9 RPM, safe for free tier).

## Usage

After `npm start`:

```
Scaler Clone Agent — chat with the agent. Type 'exit' to quit.
Try: 'clone the scaler academy website'

you > clone the scaler academy website
```

The agent will stream its reasoning and actions:

```
[THINK]  The user wants to clone the Scaler Academy website. I need to create a directory
         named `scaler-clone` first, then create index.html, style.css, and script.js inside it.
[TOOL]   createDirectory(scaler-clone)
[OBS]    Created directory scaler-clone
[THINK]  Now I will create the index.html file with the basic structure, header, hero
         section, and footer as per the requirements.
[TOOL]   writeFile({"path":"scaler-clone/index.html","content":"<!DOCTYPE html>..."})
[OBS]    Wrote 4770 bytes to scaler-clone/index.html
[THINK]  Next, I will create style.css...
[TOOL]   writeFile({"path":"scaler-clone/style.css", ...})
[OBS]    Wrote 6807 bytes to scaler-clone/style.css
[TOOL]   writeFile({"path":"scaler-clone/script.js", ...})
[OBS]    Wrote 941 bytes to scaler-clone/script.js
[TOOL]   listDirectory(scaler-clone)
[OBS]    file  index.html
         file  script.js
         file  style.css

[OUTPUT] All files created. Open scaler-clone/index.html in your browser to view the site.
```

When done, open `scaler-clone/index.html` in your browser. You can keep chatting:

```
you > add a testimonials section under the hero
you > make the header transparent on top
you > exit
```

## How the agent loop works

Each turn, the model returns a single JSON object:

```json
{ "step": "THINK", "content": "..." }
{ "step": "TOOL", "tool_name": "writeFile", "tool_args": { "path": "...", "content": "..." } }
```

The host (in [`index.js`](./index.js)):

1. Sends the conversation to Gemini with `response_format: { type: "json_object" }`.
2. Parses the JSON, appends it to history as the assistant's message.
3. If `step === "TOOL"`, looks up the tool, runs it, and pushes the result back as a synthetic `OBSERVE` user message.
4. Repeats until `step === "OUTPUT"`, then yields control back to the user prompt.

Rate-limit handling sits in `chatWithRetry`: it enforces `MIN_DELAY_MS` between calls and retries 429s with backoff (5s → 10s → 20s → 40s → 60s).
